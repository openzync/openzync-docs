repositories package¶

Repository layer — all database access lives here.

Repositories are responsible only for persistence concerns (queries, transactions, connection management). They return domain ORM models and must not contain business logic — that belongs in the service layer.

Every repository class accepts an AsyncSession in its constructor.

async get_facts_at_time(project_id, timestamp, *, organization_id=None, limit=50, offset=0)[source]¶

Return non-invalidated facts valid at a specific point in time.

Uses the btree index ix_fact_user_valid_range on (user_id, valid_from, valid_to) — not the GiST exclusion constraint index.

Query pattern:

WHERE project_id = :project_id
  AND invalid_at IS NULL
  AND (valid_from IS NULL OR valid_from <= :timestamp)
  AND (valid_to IS NULL OR valid_to > :timestamp)

LIMIT :limit OFFSET :offset

async get_facts_in_range(project_id, start, end, *, organization_id=None, limit=50, offset=0)[source]¶

Return non-invalidated facts whose valid range overlaps [start, end).

Btree-backed query (NOT GiST &&):

WHERE project_id = :project_id
  AND invalid_at IS NULL
  AND (valid_from IS NULL OR valid_from < :end)
  AND (valid_to IS NULL OR valid_to > :start)
ORDER BY valid_from ASC
LIMIT :limit OFFSET :offset
